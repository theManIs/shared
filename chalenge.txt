2. Банковский онлайн-кабинет (срок до 20.10.2015) 
При открытии кабинета доступна только страница на авторизацию. 
На вход надо поставить защиту от перебора пароля - бан по ip на 3 минуты после пятой неудачной попытки. 
После удачного входа доступна для каждого пользователя страница с информацией о состоянии счета 
и последние операции по счету. 
Также необходимо написать php страницу (к примеру api.php) через которую в зависимости от переданных 
get параметров возвращается информация. 

- без get параметров - api.php 
вернуть текст об ошибочном запросе и вывести для примера допустимые запросы 
- авторизация - api.php?action=auth&user=admin@bank.com&password=12345 
в случае успешной авторизации возвращаются данные в формате json с токеном сессии 
в случае неверных логина и пароля возвращаются данные в формате json с текстом ошибки 
- запрос состояния счета - api.php?action=balance&user=admin@bank.com&token=gjhy123h443 
возвращаются данные в формате json с содержимым баланса 
Необходимо использовать БД (Mysql).
